5. Développement de l'endpoint `GET /api/produits`

Le point d'entrée principal expose les produits avec pagination, validation de paramètres, cache et sérialisation JSON.

```php
#[Route('/api/produits', name: 'app_produits_list', methods: ['GET'])]
public function getAllProducts(
	ProduitsRepository $produitsRepository,
	SerializerInterface $serializer,
	Request $request,
	TagAwareCacheInterface $cache,
	\Psr\Log\LoggerInterface $logger
): JsonResponse {
	try {
		$page = (int) $request->query->get('page', 1);
		$limit = (int) $request->query->get('limit', 50);

		if ($page < 1) $page = 1;
		if ($limit < 1 || $limit > 500) $limit = 50;

		$contentLength = $request->server->get('CONTENT_LENGTH');
		if ($contentLength !== null && (int) $contentLength > 1048576) {
			return new JsonResponse(['error' => 'Payload too large'], Response::HTTP_REQUEST_ENTITY_TOO_LARGE);
		}

		$idCache = "getAllProduits-" . $page . "-" . $limit;

		$response = $cache->get($idCache, function (ItemInterface $item) use ($produitsRepository, $page, $limit) {
			$item->tag("produitsCache");
			$produitsList = $produitsRepository->findAllWithPagination($page, $limit);
			$total = $produitsRepository->countAll();
			$totalPages = ceil($total / $limit);

			return [
				'data' => $produitsList,
				'pagination' => [
					'page' => $page,
					'limit' => $limit,
					'total' => (int) $total,
					'totalPages' => (int) $totalPages,
				],
			];
		});

		$jsonResponse = $serializer->serialize($response, 'json');
		return new JsonResponse($jsonResponse, Response::HTTP_OK, [], true);
	} catch (\Exception $e) {
		$logger->error('Error getting products: ' . $e->getMessage());
		return new JsonResponse(['error' => 'Erreur lors de la récupération des produits'], Response::HTTP_INTERNAL_SERVER_ERROR);
	}
}
```

## 4) Ce qui est pertinent dans cette implémentation

- Validation défensive des paramètres `page` et `limit` pour éviter les valeurs incohérentes
- Borne haute sur `limit` pour limiter la charge serveur
- Cache taggé par combinaison page/limit pour accélérer les lectures répétées
- Pagination complète (`page`, `limit`, `total`, `totalPages`) directement renvoyée au client
- Journalisation des erreurs pour faciliter le diagnostic en production

## 5) Retour d'expérience

- J'ai progressé sur la structuration d'une API propre (route, repository, sérialisation)
- L'ajout du cache m'a permis de mieux comprendre les optimisations côté backend
- Les tests Postman m'ont aidé à valider rapidement le contrat de réponse
- Cette réalisation m'a donné une vision plus complète d'un flux API: base, code, sécurité, test
